# Белая страница на GitHub Pages — как проверить

Если открываете `https://budorazhka.github.io/LastDance/` и видите пустую страницу:

## 1. Откройте консоль браузера

- **Windows/Linux:** `F12` или `Ctrl+Shift+J`
- **Mac:** `Cmd+Option+J`

Вкладка **Console**. Если при загрузке есть красные ошибки (например, скрипт не найден 404, или ошибка в коде) — они и есть причина.

## 2. Проверьте вкладку Network (Сеть)

- Откройте **Network**, обновите страницу (`F5`).
- Убедитесь, что запросы к `index.html`, `index-....js` и `index-....css` в папке `/LastDance/assets/` возвращают **200**, а не 404.
- Если JS или CSS дают 404 — значит, сборка или base path настроены неверно.

## 3. Правильный адрес

Приложение с хеш-маршрутизацией. Открывайте:

- `https://budorazhka.github.io/LastDance/`
- или `https://budorazhka.github.io/LastDance/#/`

После деплоя подождите 1–2 минуты, затем обновите страницу с очисткой кэша: `Ctrl+Shift+R` (или `Cmd+Shift+R` на Mac).

## 4. Если появился текст «Ошибка загрузки приложения»

Значит, приложение запустилось, но упало при рендере. Текст ошибки на странице и сообщение в консоли (F12 → Console) подскажут, что исправить.
