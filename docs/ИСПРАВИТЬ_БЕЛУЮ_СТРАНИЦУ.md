# Что делать, если сайт белый и в консоли 404 на main.tsx

## Шаг 1. Запустить деплой заново

1. Открой на GitHub репозиторий **LastDance**.
2. Перейди на вкладку **Actions** (над списком файлов).
3. Слева выбери workflow **«Deploy to GitHub Pages»**.
4. Открой последний запуск (самый верхний).
5. Справа нажми **«Re-run all jobs»** (Перезапустить все задания).
6. Дождись, пока оба шага (**build** и **deploy**) станут зелёными (1–2 минуты).

## Шаг 2. Открыть сайт без кэша

1. Открой **новое окно в режиме инкогнито** (Ctrl+Shift+N в Chrome).
2. Вставь в адресную строку: **https://budorazhka.github.io/LastDance/**
3. Нажми Enter.

Если страница загрузилась — закрой инкогнито и открой тот же адрес в обычном окне, затем нажми **Ctrl+Shift+R** (жёсткое обновление).

## Шаг 3. Если всё ещё белая страница

В том же окне (на странице сайта):

1. Правый клик по странице → **«Просмотр кода страницы»** (View Page Source).
2. Найди строку с `<script` (примерно в начале).
3. Напиши, что там:
   - если **`/LastDance/assets/index-...js`** — сборка правильная, проблема скорее в кэше или расширениях;
   - если **`/src/main.tsx`** — тогда на сайт попал не собранный проект, нужно проверить Actions (шаг 1) и что в настройках Pages выбран источник **GitHub Actions**.

После этого можно написать сюда, что получилось на шагах 1–3.
